# 前端项目启动指南

## 📦 项目结构说明

本项目采用**多端口分离部署**方案，每个角色（教师、学生、管理员）独立运行在不同端口：

- **教师端**: http://localhost:5173
- **学生端**: http://localhost:5174  
- **管理员端**: http://localhost:5175

## 🚀 启动命令

### 1. 首次安装依赖

```bash
cd D:\project\TOS-main\web
npm install
```

### 2. 启动不同端口

#### 教师端（端口 5173）
```bash
npm run dev:teacher
# 或
npm run dev
```

#### 学生端（端口 5174）
```bash
npm run dev:student
```

#### 管理员端（端口 5175）
```bash
npm run dev:admin
```

### 3. 同时启动所有端口（推荐）

**Windows PowerShell:**
```powershell
# 打开三个终端窗口分别执行
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd D:\project\TOS-main\web; npm run dev:teacher"
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd D:\project\TOS-main\web; npm run dev:student"
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd D:\project\TOS-main\web; npm run dev:admin"
```

或者在 VS Code 中打开三个终端分别执行：
```bash
# 终端1
npm run dev:teacher

# 终端2  
npm run dev:student

# 终端3
npm run dev:admin
```

### 4. 快速自检（可选）

```bash
npm run build:teacher
```

## 🔧 环境变量

每个端口通过 `VITE_APP_ROLE` 环境变量区分角色：
- `VITE_APP_ROLE=teacher` - 教师端
- `VITE_APP_ROLE=student` - 学生端
- `VITE_APP_ROLE=admin` - 管理员端

启动后访问对应地址：

| 角色 | 端口 | 访问地址 | 登录页 |
|------|------|---------|--------|
| 教师 | 5173 | http://localhost:5173 | http://localhost:5173/teacher/login |
| 学生 | 5174 | http://localhost:5174 | http://localhost:5174/student/login |
| 管理员 | 5175 | http://localhost:5175 | http://localhost:5175/admin/login |

## 🔄 API 代理配置

所有前端端口都通过 Vite 代理转发到后端：

```javascript
// vite.config.js
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080/training-system',
      changeOrigin: true,
      rewrite: (p) => p.replace(/^\/api/, '/api')
    }
  }
}
```

**API 请求流程：**
```
前端: http://localhost:5173/api/training/cases
  ↓ (Vite 代理)
后端: http://localhost:8080/training-system/api/training/cases
```

## ⚠️ 注意事项

1. **确保后端服务已启动**: 
   ```bash
   cd <后端项目目录>
   mvn spring-boot:run
   ```
   后端地址：http://localhost:8080/training-system

2. **端口占用检查**:
   如果端口被占用，修改 `vite.config.ts` 中的端口号

3. **跨域问题**:
   已通过 Vite 代理解决，无需后端额外配置 CORS

4. **Token 管理**:
   - 自动存储在 `localStorage`
   - 请求拦截器自动添加到请求头
   - 401 错误自动跳转登录页

## 🐛 常见问题

### Q: 如何在 Windows 上设置环境变量？
A: Windows PowerShell 需要使用不同的语法：
```powershell
# PowerShell
$env:VITE_APP_ROLE="teacher"; npm run dev

# 或者使用 cross-env
npm install -D cross-env
# 然后修改 package.json
"dev:teacher": "cross-env VITE_APP_ROLE=teacher vite --port 5173"
```

### Q: 启动后页面空白？
A: 检查：
1. 路由配置是否正确
2. 组件路径是否存在
3. 控制台是否有错误
4. 后端 API 是否可访问

### Q: API 请求失败？
A: 检查：
1. 后端是否启动（http://localhost:8080/training-system）
2. 网络代理配置是否正确
3. userId 是否已设置（登录后自动设置）
